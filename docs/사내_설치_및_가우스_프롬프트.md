# 사내 PC 설치·실행 가이드 & 가우스(Gauss) 프롬프트

이 문서는 [RAG-AGENT](https://github.com/whaleonbush/RAG-AGENT) 저장소를 사내 PC(VS Code + 가우스)에서 받아 **구현 확인**할 때 사용합니다.

---

## 1. 이 프로젝트가 하는 일

- **과제(프로젝트)마다** 문서·메모를 모아 둡니다.
- PDF, DOCX, XLSX, TXT, MD와 **수동 메모**를 Markdown으로 정규화합니다.
- **청킹 → 임베딩 → Chroma**에 과제별로 저장합니다.
- 질문하면 **RAG**로 관련 구절을 찾고, **Ollama**가 있으면 자연어 답변, 없으면 검색 요약으로 답합니다.
- **별도 React 프론트는 없습니다.** Swagger(`/docs`) + 최소 HTML(`/`)로 확인합니다.

---

## 2. 사전 준비

| 항목 | 권장 |
|------|------|
| Python | 3.9 이상 (3.11 권장) |
| Git | clone용 |
| VS Code | Python 확장 |
| 네트워크 | `pip install`용 PyPI 접근 |
| (선택) Ollama | 사내 LLM 답변용 |
| (선택) Docker | `docker compose`로 Ollama 실행 |

**주의:** 첫 **인덱싱** 시 FastEmbed가 Hugging Face에서 모델을 받습니다. 사내에서 HF가 막히면 `.env`에서 `EMBEDDING_PROVIDER=ollama`로 바꾸세요.

## 3. 설치·실행 (사내 PC)

```bash
git clone https://github.com/whaleonbush/RAG-AGENT.git
cd RAG-AGENT

python -m venv .venv
# Windows: .venv\Scripts\activate
# macOS/Linux:
source .venv/bin/activate

python -m pip install --upgrade pip
pip install -e ".[dev]"

cp .env.example .env
uvicorn project_agent.main:app --reload
```

### 확인 URL

- API 문서(Swagger): http://127.0.0.1:8000/docs
- 간단 채팅 UI: http://127.0.0.1:8000/
- 헬스체크: http://127.0.0.1:8000/health

### 자동 테스트

```bash
pytest -q
```

4개 테스트가 통과하면 핵심 로직은 정상입니다.

## 4. 구현 확인 순서 (Swagger)

1. **POST /projects** — 과제 생성 → 응답의 `id` 복사  
2. **POST /projects/{project_id}/documents/manual** — 예시:
   ```json
   {
     "title": "모터 스펙 메모",
     "content": "정격 전압은 24V이며 허용 오차는 ±5%입니다.",
     "tags": ["motor"]
   }
   ```
3. **POST /projects/{project_id}/index** — 인덱싱 (첫 실행 시 모델 다운로드로 시간 소요 가능)  
4. **POST /projects/{project_id}/chat** — 예시:
   ```json
   { "question": "정격 전압이 얼마야?" }
   ```
5. 응답의 `answer`와 `citations`에 24V 관련 내용이 있는지 확인  

파일 업로드는 **POST /projects/{project_id}/documents/upload** (multipart)를 사용합니다.

### `/` UI 사용법

- Swagger에서 만든 **과제 ID**를 입력하고 질문만 보낼 수 있습니다.
- 업로드·과제 생성은 Swagger에서 하는 것이 편합니다.

## 5. Ollama (선택, 온프레 LLM)

```bash
docker compose up ollama -d
docker exec -it $(docker ps -qf name=ollama) ollama pull llama3.2
```

`.env` 예시:

```env
OLLAMA_BASE_URL=http://localhost:11434
OLLAMA_MODEL=llama3.2
```

Ollama가 없어도 **검색·인용(citations)** 은 동작합니다. `used_ollama: false`이면 검색 요약 모드입니다.

임베딩도 Ollama만 쓰려면:

```env
EMBEDDING_PROVIDER=ollama
OLLAMA_EMBED_MODEL=nomic-embed-text
```

(`ollama pull nomic-embed-text` 필요)

## 6. 사내 망에서 자주 막히는 것

| 증상 | 원인 | 대응 |
|------|------|------|
| `pip install` 실패 | PyPI 차단 | 사내 PyPI 미러 / 오프라인 wheel |
| 인덱싱 400/실패 | HF 모델 다운로드 불가 | `EMBEDDING_PROVIDER=ollama` |
| 답변이 짧은 요약만 | Ollama 미연결 | 정상(fallback). Ollama 기동 또는 무시 |
| `storage/` 없음 | 첫 실행 전 | 서버 한 번 실행하면 자동 생성 |

**GitHub에는 코드만 올립니다.** 과제 문서·벡터 DB는 `storage/`에만 생기며 `.gitignore`로 제외됩니다.

## 7. 가우스(Gauss)에 붙여넣을 프롬프트

### 7-A. 저장소 clone 후 (권장)

```
이 repo를 clone 했다: https://github.com/whaleonbush/RAG-AGENT.git

README와 docs/사내_설치_및_가우스_프롬프트.md 를 읽고,
Windows(또는 macOS) 사내 PC에서 venv 생성 → pip install -e ".[dev]" → pytest → uvicorn 실행까지 단계별로 도와줘.

에러가 나면 사내 망 제약(PyPI, HuggingFace, Docker/Ollama)을 가정하고 해결책을 제시해줘.
pytest 4개 통과와 Swagger에서 manual → index → chat 흐름이 되면 성공이다.
```

```

storage/        런타임 데이터 (git 제외)
```
